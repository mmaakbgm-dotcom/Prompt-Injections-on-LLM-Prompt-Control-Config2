CHANGE (this is what makes it “vulnerable baseline” like the papers):
A) Trust LLM-generated SQL after lightweight checks:
   - Keep the pipeline: user text → LLM generates SQL → backend validation → execute SQL.
B) REMOVE deterministic row-level enforcement in backend code:

C) Keep only lightweight backend validation that real systems often use:
   - Require authenticated session for DB-path requests.
   - Allow only SELECT statements (block INSERT/UPDATE/DELETE/DROP/ALTER/CREATE).
   - Block schema/system access attempts (PRAGMA, sqlite_master/sqlite_, information_schema).

   - Optionally enforce a hard LIMIT (e.g., LIMIT 50) on allowed queries to reduce blast radius.
D) Ensure the app still functions:

   - Out-of-scope attempts should be denied only by lightweight checks OR by the model refusing; user-facing message must still be "Request denied." if blocked.
